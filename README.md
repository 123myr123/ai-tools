# JSON Редактор - Инструкция

## 📁 Структура проекта:

```
project/
├── index.html          # HTML интерфейс с кнопкой
├── app.js              # JavaScript логика
├── data.json           # Файл данных (изменяется)
└── data/               # Папка для JSON файлов
```

## 🚀 Как запустить:

1. Откройте **index.html** в браузере
2. Нажмите кнопку "Изменить act на 1"
3. Значение `act` в `data.json` изменится с 0 → 1

## ⚠️ ВАЖНО: CORS Ограничения!

При открытии index.html локально браузер **блокирует** запросы к файлам из соображений безопасности (CORS policy). Чтобы это обойти:

### Вариант 1: Простой - используйте Python сервер
```bash
cd /путь/к/проекту
python3 -m http.server 8000
# Откройте в браузере: http://localhost:8000
```

### Вариант 2: Для production - нужен CORS middleware
Если вы используете backend (Node.js, Python Flask/FastAPI), добавьте заголовки:
- `Access-Control-Allow-Origin: *`

## 🔧 Возможности кода:

- **Чтение данных**: Fetch API загружает JSON файл при загрузке страницы
- **Запись данных**: PUT запрос обновляет файл data.json
- **Обработка ошибок**: Показывает сообщения об ошибках пользователю

## 📝 Содержимое JSON файла (data.json):

```json
{
  "act": 0,           // ← Это значение меняется при клике на кнопку
  "timestamp": "",    // Можно добавлять дополнительные поля
  "description": "JSON файл для хранения состояния"
}
```

## 💡 Примечания:

- Файл **data.json** находится в корне проекта рядом с index.html
- При каждом обновлении страницы JSON файл перезагружается заново
- Для сохранения изменений между перезагрузками используется именно локальный файл (не localStorage)

## ⚙️ Техническая информация:

Используется метод HTTP `PUT` для обновления файла. Это требует:
1. Прав доступа к файлу на сервере/файловой системе
2. Возможности backend-сервера (или браузерной среды с CORS отключенным)
3. Для локального тестирования используйте Python http.server
